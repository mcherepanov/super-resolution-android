# Политика безопасности

## Поддерживаемые версии

| Версия | Поддержка |
|--------|-----------|
| master (текущая) | да |
| старые коммиты | по возможности |

## Сообщить об уязвимости

**Не создавайте публичный issue** с описанием уязвимости.

Сообщите приватно одним из способов:

- GitVerse: личное сообщение пользователю [Max_Cherep](https://gitverse.ru/Max_Cherep);
- issue с пометкой «только для автора», если платформа позволяет ограничить видимость.

Укажите:

- описание проблемы и шаги воспроизведения;
- версию приложения / коммит;
- оценку критичности (если есть).

## Сроки ответа

- подтверждение получения — в течение 7 дней;
- исправление или план — по серьёзности, обычно до 30 дней.

## Публикация

После исправления — краткое описание в release/issue (без деталей эксплуатации до выхода патча).

## Область

- Android-приложение SR Monitor;
- сетевое взаимодействие с API сервера (учётные данные в приложении, cleartext HTTP в локальной сети).

Уязвимости backend [super-resolution](https://gitverse.ru/Max_Cherep/super-resolution) сообщайте в том репозитории.
