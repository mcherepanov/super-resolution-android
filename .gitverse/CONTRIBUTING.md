# Участие в проекте

Спасибо за интерес к SR Monitor. Проект личный, но предложения и исправления приветствуются.

## Как предложить изменение

1. Создайте issue в GitVerse с кратким описанием проблемы или идеи.
2. Сделайте fork репозитория и ветку от `master`.
3. Внесите изменения, проверьте сборку:

   ```bash
   ./gradlew assembleDebug
   ```

4. Откройте запрос на слияние (pull request) с описанием:
   - что изменено и зачем;
   - как проверить (шаги на эмуляторе/устройстве).

## Требования к коду

- Kotlin, Jetpack Compose — без лишней абстракции.
- Минимальный diff: не смешивать рефакторинг с фичей.
- Имена и структура — в духе существующего кода в `app/src/main/java/ru/max/superresolution/monitor/`.
- Перед отправкой PR: `./gradlew assembleDebug` без ошибок.

## Коммиты

Краткое сообщение на русском или английском, по смыслу:

```
fix: не запускать сервис при выключенных уведомлениях
feat: фоновый мониторинг через foreground service
```

## Ревью

Ответ на PR — по возможности в течение нескольких дней. Крупные изменения обсуждаем в issue до реализации.
